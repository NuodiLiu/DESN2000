# Prototype Test Plan

Each test maps to one Sprint 1 user story and is run on the low-fidelity Figma clickable prototype. Method: moderated usability test, 5 participants per test, think-aloud protocol.

| # | Test Goal | Related User Story | Target User | Task Scenario | Success Criteria | Expected Errors | Data to Collect | Responsible |
|---|-----------|--------------------|-------------|---------------|------------------|-----------------|-----------------|-------------|
| T1 | Verify users can create a lifestyle profile without guidance | As a student, I want to create a lifestyle profile so that other users can understand how I live | Student new to shared housing | "You just signed up. Set up your lifestyle profile (cleanliness, sleep schedule, noise, guests, smoking)." | ≥4/5 users complete the profile in ≤3 min with no facilitator help; all required fields filled | Users skip optional fields thinking they are required; users cannot find where profile setup starts; unclear field labels (e.g. "privacy") | Task completion rate; time on task; fields left blank; points where users ask for help | Pranav |
| T2 | Verify users can find a listing's key details and read its verification status | As a student, I want to see property details and the verification status of listings and landlords so that I can avoid scams | Student searching for a room | "Find a room under $350/week, check whether the listing and landlord are verified, and open the property details." | ≥4/5 users correctly state the listing's verification status; all key details (rent, fees, location, photos) found in ≤2 min | Users miss the verification badge; users confuse "listing verified" with "landlord verified"; users cannot explain what "verified" means | Verification status identification accuracy; time to locate details; user's stated meaning of the badge | Nicky |
| T3 | Verify users can compare housemate compatibility and identify the better match | As a student, I want to compare housemate preferences so that I can find a good match and reduce conflict | Student choosing between candidate housemates | "Compare two housemate profiles against your own preferences and pick the better match. Explain your choice." | ≥4/5 users pick the higher-compatibility profile and cite at least one specific mismatch (e.g. noise, smoking) | Users misread the comparison layout; users judge by photo instead of preference data; comparison entry point not found | Match selection accuracy; reasons given for choice; misclick count on comparison screen | Liam |
| T4 | Verify users can send a guided first message without sharing personal contact details | As a student, I want guided message prompts and in-app messaging so that I do not share my phone number too early | International student contacting a landlord for the first time | "You want to ask about this listing. Send a first message to the landlord using the app." | ≥4/5 users send a message using at least one guided prompt in ≤2 min; 0 users attempt to type personal contact details | Users ignore prompts and write free text; users look for a phone number instead of in-app chat; prompt wording unclear to non-native speakers | Prompt usage rate; task completion rate; whether users attempt to share contact details; comprehension issues noted in think-aloud | Dylan |

## After Testing

- Compile results in one summary: completion rates, common errors, quotes from think-aloud.
- Rank issues by severity (blocks task > causes confusion > cosmetic) and feed into the next prototype iteration.
