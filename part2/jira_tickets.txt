[ST4-23] US1.1 — Domestic Student
  Type: Story   Status: To Do   Priority: Medium
  Assignee: Unassigned
  Parent/Epic: ST4-19 Epic 1: Household Task Management
  URL: https://unsw-team-desn-seng2026.atlassian.net/browse/ST4-23
  Description / Acceptance Criteria:
    As a domestic student managing shared living for the first time, I want to set up a rotating chore roster and track completion history, so that household responsibilities are distributed fairly and I can refer back to who has and hasn't been contributing.
    Acceptance Criteria (Given/When/Then):
    Success path:
    - Given a user has created a household and added housemates, when they set up a chore roster with task assignments and a rotation schedule, then the system distributes chores evenly and notifies each housemate of their assigned tasks
    - Given a housemate has been assigned a chore, when they mark it as complete, then the system logs the completion with a timestamp and updates the participation summary
    - Given a user wants to review contributions, when they access the chore history, then they can see a full log of who completed what and when
    Error path:
    - Given a user attempts to create a roster without assigning all housemates, when they submit, then the system prompts them to assign at least one task to each housemate before activating
    - Given a chore deadline has passed without being marked complete, when the system checks for overdue tasks, then it flags the chore as overdue and notifies the assigned housemate
    Edge case:
    - Given a housemate leaves the household, when their assigned chores become unassigned, then the system alerts the house admin to reassign those tasks
    Functional Requirements:
    - The system must allow users to create, assign and schedule chores for each housemate
    - The system must automatically rotate chore assignments on a defined schedule
    - The system must log chore completions with timestamps and display a participation summary
    - The system must flag overdue chores visually and notify the assigned housemate
    Non-Functional Requirements:
    - The chore roster must load within 2 seconds on a standard mobile connection
    - The system must support households of up to 10 members
    - Chore history must be retained for the duration of the tenancy
